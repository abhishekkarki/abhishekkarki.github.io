Reading

Books and papers worth your time

Things I've read that changed how I think about building systems. Ordered by how often I find myself referencing them, not by when I read them.

Essential Books

  • Designing Data-Intensive Applications — Martin Kleppmann. The single best book on distributed systems for practitioners.
  • Database Internals — Alex Petrov. Storage engines, B-trees, LSM trees, and distributed algorithms from first principles.
  • Systems Performance — Brendan Gregg. How to actually understand what a system is doing under the hood.
  • The Art of Unix Programming — Eric Raymond. Old but the philosophy holds. Small tools, composability, text as interface.
  • A Philosophy of Software Design — John Ousterhout. The best short book on managing complexity in code.

Papers

  • Dynamo: Amazon's Highly Available Key-value Store — the eventual consistency paper that started many conversations
  • In Search of an Understandable Consensus Algorithm (Raft) — Ongaro & Ousterhout
  • Bigtable: A Distributed Storage System for Structured Data — the foundation for HBase, Cassandra, and many others
  • A Relational Model of Data for Large Shared Data Banks — Codd. Worth reading the original.
  • Spanner: Google's Globally-Distributed Database — TrueTime and external consistency

Blogs Worth Following

  • Martin Kleppmann — CRDTs, distributed systems
  • Marc Brooker — AWS systems engineering, formal methods
  • Julia Evans — systems concepts made accessible
  • Rachel by the Bay — production horror stories and lessons
  • Brendan Gregg — performance engineering
